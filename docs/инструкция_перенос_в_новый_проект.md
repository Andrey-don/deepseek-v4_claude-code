# Как перенести связку DeepSeek V4 + Claude Code в новый проект

## Что установлено глобально (один раз)

Все инструменты установлены на машине и работают из любой папки:

| Команда | Что делает |
|---------|------------|
| deepseek | DeepSeek TUI — терминальный интерфейс DeepSeek V4 Pro |
| claude | Claude Code Sonnet 4.6 — оркестратор |
| claude-ds | Claude Code с DeepSeek под капотом (нужен fcc-server) |
| fcc-server | Прокси-сервер для claude-ds |

Переустанавливать для каждого нового проекта не нужно.

---

## Что нужно для нового проекта

Только одно — скопировать CLAUDE.md в корень нового проекта.

Этот файл содержит инструкцию для Claude: когда делегировать задачи DeepSeek и как это делать.

---

## Алгоритм запуска в новом проекте

1. Открой новый проект в VS Code
2. Скопируй CLAUDE.md из шаблонного репозитория в корень проекта
3. Открой терминал: Ctrl+backtick
4. Верхний терминал: deepseek
5. Split Terminal -> нижний терминал: claude
6. Работай — Claude сам решает кому отдать задачу

---

## Быстрый старт через клонирование шаблона

Клонировать этот репозиторий как основу нового проекта:

    git clone --recurse-submodules https://github.com/Andrey-don/deepseek-v4_claude-code.git название-проекта

Внутри уже есть CLAUDE.md, SETUP.md, free-claude-code, docs/.

## Три режима работы

### Режим 1 — Claude оркестратор + DeepSeek исполнитель

Верхний терминал: deepseek | Нижний терминал: claude
Claude читает CLAUDE.md и делегирует рутину DeepSeek командой: deepseek задача

### Режим 2 — DeepSeek оркестратор + Claude исполнитель

Верхний терминал: deepseek | Нижний терминал: claude
В начале сессии сказать DeepSeek: Сложные задачи делегируй Claude командой: claude -p задача

### Режим 3 — Только DeepSeek

Один терминал: deepseek. Максимальная экономия.

---

## Когда что использовать

| Режим | Когда |
|-------|-------|
| 1 — Claude ведёт | Средние и сложные проекты |
| 2 — DeepSeek ведёт | Большие проекты с потоком задач |
| 3 — Только DeepSeek | Рутина, рефакторинг, тесты, документация |
